## Лабораторная работа №2. Применение Elasticsearch в приложении Todo.

### Отчёт
Личная демонстрация работы макета + результаты в личном репозитории + электронный отчёт (создаёте issue с названием "отчёт ЛР2", где оформляете задание, ссылку на свой код, скриншоты результатов).

### Задание

Предполагается командная работа в рамках команды (2-4 человека). Задачи ниже должны быть декомпозированы и распределены между разработчиками равномерно. Список команды передать преподавателю (fullname участников с отметкой, кто maintainer). Работать в проекте-тудушке res/otip/<year>/fullname у maintainer проекта. Продолжаем развивать проект fastapi-todo. К коду должны быть написаны тесты с покрытием не менее 60%.

1. Проверить, что есть поле для ввода числа тудушек и кнопку "сгенерировать". В результате должно быть сгенерировано указанное число тудушек со случайными заголовками.

1. Проверить систему аутентификации (JWT) и разграничения прав доступа, наличие регистрации пользователя и наличие пользователя user с паролем user.

1. Реализовать агрегацию, то есть группировку заметок по дням/неделям/месяцам.

1. Реализовать визуализацию активности разных пользователей в виде графика или диаграммы - число заметок в день от пользователя. Можете оттолкнутся от логики ниже

    ```json
    {
    "aggs": {
        "notes_per_day": {
        "date_histogram": {
            "field": "created_at",
            "calendar_interval": "day"
        }
        }
    }
    }
    ```

1. Реализовать привязку заметок к пользователям (только автор может редактировать/удалять). (ДЕЛАЮ СЕЙЧАС, работает только создание)

1. Реализовать отдельную роль admin для управления всеми заметками.

1. Реализовать историю изменений заметки (кто и когда редактировал).

1. Вместо фиксированных тегов реализовать произвольные теги, которые сохраняются в эластике. Реализовать создание тега, удаление тега, назначения тега при создании тудушки. При создании тега подсказывать уже существующие теги (вроде автодополнения, чтобы не создавать ошибочно теги типа "учёба" и "учеба", отличающиеся незначительно).
